In Git, a branch is like a timeline. Each branch contains a sequence of snapshots. Each of those snapshots is a single event in the branch‚Äôs history.

Open a terminal in your clone‚Äôs directory, and run this command:

Git will then tell you what branch you‚Äôre currently on. It will also list any other branches that you add to your clone.   1. Determine what branch you‚Äôre currently on: 
    git branch
  2. If you‚Äôre not on the default branch, the switch to it: 
    git checkout main

    If the repo doesn‚Äôt have a branch named ‚Äúmain‚Äù, then adjust that command accordingly.

  3. Create a copy of your current branch: 
    git branch <branch-name>
  4. Switch to your new branch: 
    git checkout <branch-name>

At this point, you‚Äôve created your own branch but haven‚Äôt added anything to it yet. At the moment, your branch is identical to the main branch. Now it‚Äôs time to get to work. Create new files, delete old files or edit existing files. Don‚Äôt worry too much about making an unwanted change because Git is already holding onto multiple snapshots of the files in the repo. Every so often, you should create one of these snapshots. To do so:

  1. Determine what files have been created, deleted or edited: 
    git status
  2. Before you can take a snapshot, you have to ‚Äústage‚Äù your changes. Changes that aren‚Äôt staged won‚Äôt make it into the snapshot.
    • If you created or edited a file, then run: 
      git add <path-to-file>
    • If you deleted a file, then run: 
      git rm <path-to-file>
    • If you renamed or moved a file, then run: 
      git mv <old-path> <new-path>
    You can run git status again to see what changes are staged and unstaged.
  3. Take the snapshot: 
    git commit
    This will create a new commit and put it at the tip of your current branch. In Git, snapshots are called ‚Äúcommits‚Äù.
